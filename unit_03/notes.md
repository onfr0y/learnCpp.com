## 💡 Key Concepts & Best Practices

### 1. Debugging Process
- **Static Analysis**: Compile-time check using warnings (e.g. `-Wall -Wextra -Wpedantic`).
- **Instrumenting Code**: Temporary prints (`std::cout`) to watch program state.
- **Using a Debugger**: Setting breakpoints, stepping (`step over`, `step into`, `step out`), inspecting variables, and reviewing the call stack.

### 2. Call Stack
The call stack tracks active function calls. Examining it helps pinpoint the execution path leading to a bug or crash.
