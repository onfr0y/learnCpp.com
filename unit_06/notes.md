## 💡 Key Concepts & Best Practices

### 1. Operator Precedence and Associativity
- **Precedence**: Determines the order in which operators are evaluated when they have different precedence levels (e.g. multiplication `*` has higher precedence than addition `+`).
- **Associativity**: Determines the evaluation direction (left-to-right or right-to-left) when operators have the same precedence.
- **Rule**: If in doubt, use parentheses `()` to explicitly clarify precedence. It prevents bugs and makes the code self-documenting.
